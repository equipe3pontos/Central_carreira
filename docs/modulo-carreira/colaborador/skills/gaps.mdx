# Gaps (`/app/perfil` · Skills)

<AnaliseProduto>

### Finalidade

A aba **Gaps** agrega **lacunas** entre nível **atual** e **esperado**, derivadas do [radar](./matriz), das [Hard Skills](./hard), das [Soft Skills](./soft) e da análise cruzada no protótipo. Cada linha deve poder originar uma **meta no `/app/pdi`**, fechando o circuito diagnóstico → compromisso.

### Regra de produto

No mock, **13** gaps são contabilizados em conjunto com o cartão «Ver todos os gaps» na [Visão geral](../perfil-visao-geral). Estados **«Vinculado ao PDI»** vs **apenas «Criar Meta PDI»** distinguem lacunas já associadas a uma meta de desenvolvimento das que ainda não têm dono no plano.

### Rastreabilidade

| Origem | Uso |
|--------|-----|
| `/app/pdi` · [PDI](../perfil-pdi) | Metas e *work items* concretos do plano |
| Visão geral | Resumo dos gaps prioritários em cartões |
| Hard / Soft / Radar | Derivação das transições de nível |

### Limitações do protótipo

Prioridades e vínculos ao PDI são **dados de demonstração**; regras de negócio para promoção ou obrigatoriedade de meta não são definidas nesta página.

### Matriz de competências · framework L3–L5 (contexto na aba)

Este **framework** conecta as **7 competências-chave (Radar)** às **Hard Skills** técnicas e **Soft Skills** comportamentais, identificando **Gaps** que direcionam o **PDI** e a progressão de carreira **L5 → L6**. Cada competência é avaliada de forma estruturada para mapear o desenvolvimento individual.

**Separadores no mock:** Competências · Hard Skills · Soft Skills · **Gaps** — inventários relacionados: [Matriz](./matriz) · [Hard Skills](./hard) · [Soft Skills](./soft).
