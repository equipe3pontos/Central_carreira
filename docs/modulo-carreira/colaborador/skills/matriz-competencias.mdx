# Skills — matriz de competências

> Uma régua única. 7 competências (Design, Estratégia, Arquitetura, Performance, Comunicação, Trabalho Equipe, Liderança). Cada uma: escala 0-5, desdobra em hard/soft skills, conecta a L5 Senior ou L6 Staff. Atual vs Esperado = 13 gaps acionáveis.

---

## 7 competências que definem L5

Cada uma diferencia L4 de L5 de L6. Todas usam escala 0-5 (Iniciante → Especialista).

| # | Competência | O que é | Desdobra em |
|---|-------------|---------|------------|
| 1️⃣ | 🏗️ **Design de Soluções** | Arquitetura de features, padrões React, composição, trade-offs | React, TypeScript, padrões (Hard); pensamento crítico (Soft) |
| 2️⃣ | 📈 **Estratégia Técnica** | Roadmap, débito técnico, decisões 6-12 meses adiante | Next.js, SSR, AWS, CI/CD (Hard); visão, decisão (Soft) |
| 3️⃣ | 🎨 **Arquitetura Frontend** | Estrutura de componentes, escalabilidade, Design Systems | Redux, Zustand, Design Systems, micro frontends (Hard) |
| 4️⃣ | ⚡ **Performance & Otimização** | Web Vitals, bundle, rendering, acessibilidade | Lighthouse, WCAG, lazy loading, testes (Hard) |
| 5️⃣ | 💬 **Comunicação Técnica** | Apresentações claras, RFCs, alinhamento com não-técnicos | Documentação, tech talks (Hard); clareza, influência (Soft) |
| 6️⃣ | 🤝 **Trabalho em Equipe** | Colaboração cross-squad, pair programming, code review | Git, code review, pair sessions (Hard); colaboração (Soft) |
| 7️⃣ | 👨‍💼 **Liderança Técnica** | Mentoria de juniors, decisões técnicas documentadas, influência | RFCs, ADRs, decisões (Hard); mentoria, visão (Soft) |

## Escala (0-5) — todos usam mesma régua

| Nível | Você consegue | Exemplo |
|-------|--------------|---------|
| **0 — Iniciante** | Teoria, sem produção | Leu sobre RxJS; nunca implementou |
| **1 — Desenvolvendo** | Feature básica, com ajuda | React feature com review detalhado |
| **2 — Sólido** | Feature independente, know trade-offs | Entrega React sem review funcional |
| **3 — Referência** | Arquitetura maior, mentoria, padrões | Define padrões da squad; tech talk |
| **4 — Especialista** | Referência externa, publica, open source | Reconhecido em comunidade |

L5 senior fica em **Sólido-Referência** (2-3). L6 staff fica em **Referência-Especialista** (3-4).

## Exemplo: L5 Senior Frontend

Matriz Atual vs Esperado mostra gaps (diferença = oportunidade).

| Competência | Atual | Esperado | Gap | Prioridade |
|-------------|-------|----------|-----|-----------|
| 🏗️ Design Solução | Sólido | Referência | ↑ 1 | 🔴 Alta |
| 📈 Estratégia Técnica | Desenvolvendo | Sólido | ↑ 1 | 🔴 Alta |
| 🎨 Arquitetura Frontend | Sólido | Sólido | ✓ | 🟢 OK |
| ⚡ Performance | Sólido | Sólido | ✓ | 🟢 OK |
| 💬 Comunicação | Sólido | Referência | ↑ 1 | 🟡 Média |
| 🤝 Trabalho Equipe | Sólido | Referência | ↑ 1 | 🟡 Média |

- Forte em: Design, Arquitetura, Performance (Sólido).
- Crítico: Estratégia + Liderança Técnica (↑ 1 nível cada).
- Opcional: elevar Comunicação + Trabalho Equipe para Referência (desejável para L6).

## De competência para gaps acionáveis

Cada competência desdobra-se em hard/soft skills. Cada gap identifica hard ou soft específica.

**Exemplo:**
- Gap "React Advanced Patterns (Sólido → Referência)" → hard skill de Design de Soluções
- Gap "Tomada de Decisão Técnica" → soft skill de Liderança Técnica
- Gap "Web Performance" → hard skill de Performance & Otimização
