# Skills — chegar ao gap certo

> Colaborador identifica seu gap crítico em menos de 3 min. Sem matriz: "melhore comunicação" (vago). Com matriz: "TypeScript Sólido vs Referência esperado = gap claro" + ação no PDI.

---

## O problema: autodiagnóstico falha

Sem modelo estruturado:
- Feedback vago ("melhore comunicação técnica" — mas o quê especificamente?)
- Sem priorização (28 hard skills + 6 soft skills = 34 potenciais gaps)
- Soft skills abstratos (sem critérios SMART observáveis)
- Sem benchmark (bom em React? Comparado a quem?)

**Resultado:** Gap parece opinião solta. Metas PDI vagas. Taxa de conclusão 30% menor que metas SMART.

---

## Solução: matriz Atual vs Esperado (7 competências)

### As 7 dimensões

| # | Competência | Papel | Desdobra em |
|---|-------------|-------|-----------|
| 1️⃣ | 🏗️ **Design de Soluções** | Arquitetar soluções, trade-offs | React, TypeScript, padrões, composição |
| 2️⃣ | 📈 **Estratégia Técnica** | Roadmap técnico, débito, priorização | Next.js, SSR, AWS, CI/CD, monitoramento |
| 3️⃣ | 🎨 **Arquitetura Frontend** | Estrutura de componentes, escalabilidade | Redux, Zustand, Design Systems, Micro Frontends |
| 4️⃣ | ⚡ **Performance & Otimização** | Web Vitals, bundle, acessibilidade | Lighthouse, WCAG, lazy loading, testes |
| 5️⃣ | 💬 **Comunicação Técnica** | Apresentações claras, RFCs, alinhamento | Documentação, tech talks, influência |
| 6️⃣ | 🤝 **Trabalho em Equipe** | Colaboração cross-squad, pair programming | Git, code review, multiplicação |
| 7️⃣ | 👨‍💼 **Liderança Técnica** | Mentoria, decisões estruturadas, visão | RFCs, decisões técnicas, influência |

### Escala: iniciante → especialista

Todos usam mesma régua (0-5):

- **0 = Iniciante:** Teórico; sem prática produção.
- **1 = Desenvolvendo:** Prática básica; com guidance.
- **2 = Sólido:** Entrega independente, conhece trade-offs.
- **3 = Referência:** Referência interna; mentora; identifica padrões complexos.
- **4 = Especialista:** Referência externa; publica; define padrões.
