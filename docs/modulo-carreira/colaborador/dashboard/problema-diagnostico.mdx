# Dashboard — problema e diagnóstico

> Colaborador abre 5 módulos para responder uma pergunta simples. Leva 7-8 minutos. Depois sai. Com Dashboard: 2 minutos, uma tela, próximos passos visíveis.

---

## Diagnóstico: por que contexto fragmentado mata recorrência

Sem hub único, cada resposta fica em silos diferentes:

- Squad ativo: abre Perfil
- Próximos passos: abre PDI
- Feedback recente: abre Pulse
- Épico em andamento: abre Projetos
- Status do time: abre Squads

5 navegações, 7-8 minutos até contexto mínimo. Resultado: abandono.

### O custo real de contexto fragmentado

| Situação | Impacto |
|----------|---------|
| **Tempo até primeira decisão** | 5+ min contexto + 2-3 min análise = 7-8 min. Utilizador abre browser, vê complexidade, fecha. |
| **Recorrência** | ~1x/semana (só na avaliação). Deveriam ser diárias (feedback, épico, próximos passos). |
| **Abandono** | PDI criado em Q1; não volta até Q2 (avaliação). Sem toque semanal, plano morre. |
| **Feedback invisível** | Feedback recebido fica isolado em Pulse. Não conecta com carreira, desenvolvimento, próximas ações. |

## Por que fragmentação mata o loop de aprendizagem

**1. Sem hub, utilizador não sabe por onde começar**  
Cada módulo é isolado. Dados de acesso: `/app/perfil` é 3x mais visitado que `/app`. Colaborador vai direto para módulo conhecido, não para contexto integrado.

**2. Métricas invisíveis**  
Feedback, OKR, próximos passos vivem em silos. Colaborador não vê conexão: feedback recebido → impacto no PDI → próxima ação. Resultado: feedback recebido fica inerte.

**3. Progresso invisível**  
Sem timeline, não há senso de movimento. "Você recebeu feedback", "sua meta subiu", "épico iniciou" — esses sinais não aparecem. Sem movimento visível, abandono.

**4. Trabalho desconectado de carreira**  
Épico ativo em Projetos, PDI em módulo separado. Colaborador não vê como "término da feature X" conecta com "evolução para L6". Entrevistas mostram: "Não vejo relação entre meu trabalho e meu plano pessoal."

## Como produtos de carreira resolvem isso

GitHub, Jira, LinkedIn, Notion todos fizeram a mesma escolha: **hub visual + timeline + ações rápidas** como entrada principal. Por quê?

1. **Reduz fricção:** Um clique até contexto, não 5.
2. **Oferece foco:** Próximos passos visíveis eliminam paralisia.
3. **Conecta domínios:** Uma tela une trabalho (épico), aprendizagem (certificação), reconhecimento (feedback).
4. **Motiva continuidade:** Timeline de progresso oferece senso de movimento — colaborador volta.

---

## Métrica que prova sucesso

Se Dashboard reduz tempo até ação para 2 minutos e oferece próximos passos visíveis:
- Recorrência semanal (vs mensal hoje)
- Feedback loop contínuo (feedback → ação → PDI)
- Colaborador volta diariamente

**Sucesso = 70% das sessões geram ação rápida ou movem próximo passo.**
