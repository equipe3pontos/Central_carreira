## Cabeçalho global e comportamento

Os elementos do *chrome* superior aparecem em todas as telas da trilha Colaborador. O protótipo mostra intenção visual, mas nem todos os controles são interativos.

| Elemento | Comportamento observado |
|----------|-------------------------|
| **Logo 3pts** | Clicável; volta para o Dashboard (`/app`). |
| **Menu de navegação** | Mostra destaque da página ativa. |
| **Data atual** | Texto estático: `sexta-feira, 24 de abril de 2026`. |
| **Sino de notificações** | Badge vermelho de não lidas; no protótipo não abre painel. |
| **Troféu / conquistas** | Presente, mas sem ação. |
| **Avatar + nome** | Não abre dropdown no mock. |
| **Logout** | Ícone presente no header; ação final não está detalhada. |

**Regra de leitura:** estes elementos pertencem ao *shell* da aplicação, não ao fluxo principal do Dashboard. O conteúdo funcional continua sendo o centro da página.

## Checklist: alinhamento antes de produção

| Decisão | O que resolver |
|---------|---------------|
| 🎨 **Cores** | Semântica clara (não depender só de cor, A11y). Taxonomia de eventos para feed. |
| 📅 **Datas** | Mock usa Q1 2026; produção usa data real. |
| 📝 **Vocabulário** | "Feedback 360°" vs "Pulse" — unificar. |
| 🎯 **Épico** | "Autenticação & Autorização" vs "OAuth 2.1" — uma fonte de verdade. |
| 📊 **OKRs / Certs** | Origem dos dados (API/sistema), regra de atualização. |
