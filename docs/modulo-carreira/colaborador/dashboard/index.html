Especificação de Produto: Dashboard do Colaborador (/app)

O Dashboard opera como um roteador de atenção. Projetado para sessões curtas, ele deve garantir que o usuário acesse a plataforma, compreenda seu contexto imediato, tome uma decisão e navegue para a tela de destino específica.

A métrica de sucesso (North Star) desta interface é o Tempo até a Primeira Ação (Time to First Action). O Dashboard falha se for utilizado para leitura passiva prolongada.

1. Visão de Produto: Promessa de Experiência e Decisões​

O design foca em responder a perguntas implícitas do usuário em uma única varredura visual. A interface entrega respostas que destravam ações específicas.

2. Arquitetura da Informação e Carga Cognitiva​

A estrutura da página aplica o princípio de "Leitura por Camada". A informação é hierarquizada da menor para a maior carga cognitiva, guiando o olhar do macro (situação) para o micro (ação e histórico).

3. Arquitetura de Software: Modais e Limites de Contexto (Bounded Contexts)​

O Dashboard atua como uma camada de agregação (BFF - Backend for Frontend), consumindo dados de múltiplos domínios para montar a visualização inicial.

4. Jornada de Uso e Heurísticas Aplicadas​

O fluxo da tela respeita rigorosamente as heurísticas de usabilidade de Nielsen, minimizando erros e frustrações durante o acesso rotineiro.

Chegada e Reconhecimento: O usuário identifica seu contexto (saudação, alocação) sem depender da memória de sessões anteriores.

Avaliação de Estado: Visibilidade clara do status do sistema através de métricas de progresso e cartões de ciclo atual.

Decisão: A interface utiliza linguagem do mundo real (Sprint, Épico, 1:1, PDI), facilitando o pareamento entre o modelo mental do colaborador e as ações disponíveis.

Execução Segura: Ações rápidas indicam destino claro (Prevenção de Erros). Componentes distintos possuem rótulos inequívocos (Consistência entre Feedback Contínuo e Avaliação Formal).

Saída: O usuário abandona o Dashboard em direção ao fluxo profundo (PDI, Pulse, Projetos) para concluir sua tarefa de evolução.

5. Especificação Detalhada por Região​

Barra Superior​

Contexto navegacional: Logo, menu (Dashboard ativo), data, notificação, troféu, perfil, logout. Permanece visível para voltar a qualquer módulo sem perder referência.

Identificação de Squad​

Uma linha identificando squad ativa, produto, status (verde = ativo). Responde "Onde estou?" em 1 segundo.

Cartões de Métricas (5 cartões)​

Sinais rápidos de situação. Alocação (Squad + status), Feedback 360° (Recebidos/Enviados + ciclo), OKRs (progresso + período), Sprints (contagem ativa), Certificações (total obtidas).

Coluna Esquerda: Ações Rápidas​

Quatro botões de ação com destinos previsíveis: Dar Feedback, Atualizar PDI, Check-in Semanal, Daily Standup. Reduz fricção e não força navegação.

Coluna Esquerda: Épico em Andamento​

Mostra épico ativo da sprint com ID, projeto, sprint, prioridade, progresso percentual, stories, story points, ETA. Liga técnico a carreira.

Coluna Direita: Atividades Recentes​

Timeline inversa (recente no topo). Cada evento: indicador de cor, texto, timestamp ISO 8601, destino clicável. Responde "O que mudou?"

Próximos Passos (3 cartões)​

Estados definidos: PENDENTE (Amarelo) — Self-Assessment com prazo; EM ANDAMENTO (Azul) — Certificação com percentual; AGENDADO (Verde) — 1:1 com horário. Define foco e visibilidade de pendências.

6. Métricas de Sucesso​

North Star: Tempo até a Primeira Ação (TTFA).

  • Colaborador identifica ação em menos de 2 minutos.
  • 70% das sessões geram ação concreta (PDI, feedback, check-in, navegação).
  • Recorrência semanal para usuários L4+.
  • Feedback do usuário: clareza acima de 4/5.

Próximo passo​

Perfil: Entender onde você está (squad, alocação, skills, histórico).
