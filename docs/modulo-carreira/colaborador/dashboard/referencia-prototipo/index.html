Dashboard — referência do protótipo

Visual inventory of /app (Colaborador dashboard). Exatamente como aparece no protótipo 3pts: copy, cores, campos, layouts. Discovery tool, não contrato de produção. Números/datas mudam. Estrutura e semântica valem.

O que você vê na tela​

Imagem completa: static/img/dashboard-jornada-colaborador.png

Estrutura visual em 7 regiões:

RegiãoO que contém
Barra superior3pts logo · módulos (Dashboard ativo, Perfil, PDI, Pulse, Projetos, Squads) · data · notificação sino · troféu conquistas · avatar + nome (Marcelo) · logout
Boas-vindasSaudação + subtítulo (copy na tabela abaixo)
KPIs5 cartões: Alocação, Feedback 360°, OKRs, Sprints, Certificações
Coluna esqAções rápidas (4 botões) + Epic em andamento (progresso, stories, ETA)
Coluna dirAtividades recentes (timeline com cores por tipo de evento)
Rodapé3 próximos passos (estados: PENDENTE/EM ANDAMENTO/AGENDADO com cores)

Checklist: alinhamento antes de produção​

DecisãoO que resolver
CoresSemântica clara (não depender só de cor, A11y). Taxonomia de eventos para feed.
DatasMock usa Q1 2026; produção usa data real.
Vocabulário"Feedback 360°" vs "Pulse" — unificar.
Épico"Autenticação & Autorização" vs "OAuth 2.1" — uma fonte de verdade.
OKRs / CertsOrigem dos dados (API/sistema), regra de atualização.
