Mapa de rotas do protótipo

O site em reveal-essay-77974683.figma.site corresponde ao mockup exportado. A lista completa de caminhos não está apenas na documentação antiga: está no código — Módulo de Carreira (2)/src/app/routes.tsx (createBrowserRouter). Qualquer rota nova ou renomeada deve reflectir-se primeiro nesse ficheiro e depois nesta página (docs-as-code).

Escopo da entrega atual

Nesta fase, a entrega e a documentação operacional cobrem somente o Módulo Colaborador (/app/*). As rotas RH Tech People (/hr/*) continuam listadas porque existem no protótipo, mas ficam como referência de descoberta / roadmap. Não devem gerar backlog de implementação, critérios de aceite ou testes de produção agora.

Colaborador em números (só trilha /app)​

No routes.tsx, o bloco path: "app" define todas as URLs do módulo Colaborador. Não há sub-rotas escondidas noutro ficheiro — o inventário abaixo fecha com o código.

MétricaQuantidadeNotas
Rotas canónicas /app…8Uma por URL distinta: índice /app + 7 path filhos (perfil, pdi, pulse, pulse/received, pulse/sent, projetos, squads).
Aliases legados → Colaborador7Redirecionam para o mesmo conteúdo que /app/perfil … /app/squads — não existe alias para o dashboard em / (o dashboard é só em /app).
Superfície URL útil para Colaborador158 canónicas + 7 aliases (para testes e capturas, cada alias dispara um replace para a canónica).

Aliases legados (redirecionamento)​

Implementados como rotas à raiz que montam RedirectToApp: em tempo de execução navegam para /app + o mesmo pathname (replace).

Rota antigaDestino após redirecionamento
/perfil/app/perfil
/pdi/app/pdi
/pulse/app/pulse
/pulse/received/app/pulse/received
/pulse/sent/app/pulse/sent
/projetos/app/projetos
/squads/app/squads

Estas 7 URLs não são páginas diferentes do ponto de vista de produto — são compatibilidade de marcadores ou links antigos sem o prefixo /app.
